The Story

Most devices on the Internet use the TLS protocol to protect their data in transit. Unfortunately, most TLS implementations are written in C and have a long history of memory safety vulnerabilities.

It's important for the security of the Internet that we move away from TLS libraries that aren't memory safe. Since 2021 we've invested heavily in a memory safe implementation of TLS called Rustls. Today, not only is Rustls memory safe, it's fully featured while significantly faster and more efficient than most other TLS implementations.

While Rustls is popular in the Rust community, we work hard to make sure it is easy to use from existing C and C++ programs. In addition to the native Rust API, Rustls has a C API and an OpenSSL compatibility layer. For some projects, Rustls can act as a drop-in replacement for OpenSSL.
